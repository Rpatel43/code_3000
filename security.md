Intended users of repo: Ronit Patel and Prof. Lamoureux

Security threats: Injection of malicious code, an attacker with access to insert files and code could add in backdoors or malicious scripts.

Steps taken to secure repo: Implemented a rule set for pull requests. Restricted direct pushes, no user can push code directly to the main branch. Mandatory code owner reviews, requires a review from a code owner. Prevention of branch deletion, ensures primary history of the repo can't be deleted.
